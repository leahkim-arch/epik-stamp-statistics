# GitHub Personal Access Token 생성 가이드

## 단계별 방법

### 1단계: Note (이름) 입력
- **"Note"** 필드에 토큰 이름을 입력하세요
  - 예: `epik-stamp-push` 또는 `stamp-dashboard-deploy`
- 현재 빨간색 에러 메시지("Note can't be blank")가 보이면 이름을 입력해야 합니다

### 2단계: Expiration (만료 기간) 선택
- 드롭다운에서 만료 기간을 선택하세요
  - 현재: "30 days (Mar 25, 2026)"로 설정되어 있음
  - 원하는 기간 선택 (30일, 60일, 90일 등)

### 3단계: Select scopes (권한 선택) ⭐ 가장 중요!
- **"repo"** 체크박스를 찾아서 **체크**하세요
  - "Full control of private repositories"라고 설명되어 있습니다
  - 이 권한이 있어야 코드를 푸시할 수 있습니다

### 4단계: Generate token (토큰 생성)
- 페이지 맨 아래로 스크롤하세요
- **"Generate token"** (또는 "Create token") 버튼을 클릭하세요

### 5단계: 토큰 복사 ⚠️ 중요!
- 토큰이 생성되면 **즉시 복사**하세요
- `ghp_` 또는 `github_pat_`로 시작하는 긴 문자열입니다
- **이 페이지를 벗어나면 다시 볼 수 없습니다!**
- 안전한 곳에 저장해두세요

## 생성 후 사용 방법

토큰을 생성하셨다면:
1. 터미널에서 `git push -u origin main` 실행
2. Username: `leahkim-arch` 입력
3. Password: **복사한 토큰을 붙여넣기** (비밀번호가 아닙니다!)

또는 토큰을 알려주시면 제가 푸시를 진행하겠습니다!
